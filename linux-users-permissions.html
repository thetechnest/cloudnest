Core Concepts (Read Before Practice)

Linux is a multi-user operating system. Every action is performed by a user, controlled by permissions, and executed as a process. Understanding these concepts is mandatory for Hadoop, Cloud, and DevOps.

  • User ‚Äì Who is performing the action
  • Group ‚Äì Collection of users
  • sudo ‚Äì Temporary admin power
  • su ‚Äì Switch user
  • SSH ‚Äì Secure remote access
  • Permissions ‚Äì Who can read/write/execute
  • Process ‚Äì A running program
üí° Hadoop failures usually come from permission or user mistakes, not Hadoop itself.

Users & Groups

whoami

whoami

Shows the current logged-in user.

id

id

Displays user ID, group ID, and groups.

groups

groups

Shows which groups the user belongs to.

sudo and su

sudo

sudo apt update

Runs a command with administrator privileges.

su

su - hduser

Switches to another user.

SSH ‚Äì Secure Shell

ssh localhost

ssh localhost

Connects to the local machine securely.

ssh user@ip

ssh ubuntu@3.110.x.x

Connects to a remote cloud VM.

Permissions: chmod & chown

chmod

chmod 755 file.sh

Sets read/write/execute permissions.

chown

sudo chown hduser:hadoop file.txt

Changes file owner and group.

Processes: ps, grep, kill

ps -ef

ps -ef

Lists all running processes.

grep

ps -ef | grep java

Filters output to show Java processes.

kill

kill 1234

Stops a process using PID.

nano ~/.bashrc

echo

hadoop classpath

Shows Hadoop Java classpath.
