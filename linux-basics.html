Linux Basics ‚Äì Foundation Stone for Hadoop & Cloud

What is Linux?

Linux is an open-source operating system that powers servers, cloud platforms, supercomputers, and big data systems. Unlike Windows, Linux is command-line driven, giving users complete control over the system.

üí° Fact: AWS, GCP, Azure, Hadoop, Spark, and AI platforms all run on Linux.

Why Should I Learn Linux?

Learning Linux is not optional if you want to work with cloud and big data. Cloud servers do not provide mouse-based interfaces; instead, you manage everything through Linux commands.

üöÄ Linux turns you from a user into a system controller.

Linux and Cloud (AWS / GCP)

When you launch an EC2 instance in AWS or a VM in GCP, you are given a Linux machine. Installing Java, Hadoop, configuring SSH, and running MapReduce jobs all happen through Linux.

How Linux Helps You Become an AI/ML Engineer

AI/ML tools such as TensorFlow, PyTorch, Spark, and Hadoop are built and optimized for Linux environments. Linux allows you to manage data pipelines, models, and compute resources efficiently.

  • Understand commands, not memorize

Essential Linux Commands (Beginner ‚Üí Mid Level)

These commands form the daily working toolkit of Linux users. Practice them repeatedly on AWS EC2 Ubuntu until they feel natural. Mastering these commands will make Hadoop and Cloud experiments easy.

1. pwd ‚Äì Where am I?

Command: pwd

Output: /home/ubuntu

Explanation: Displays the current working directory.

2. ls ‚Äì List files

Command: ls

Output: file.txt data test

Explanation: Lists files and folders in the current directory.

3. ls -l ‚Äì Detailed listing

Command: ls -l

Output: -rw-r--r-- ubuntu ubuntu 1200 file.txt

Explanation: Shows permissions, owner, size, and timestamp.

4. cd ‚Äì Change directory

Command: cd data

Output: (no output)

Explanation: Moves into another directory.

5. cd .. ‚Äì Move one level up

Command: cd ..

Explanation: Navigates to the parent directory.

6. mkdir ‚Äì Create directory

Command: mkdir logs

Explanation: Creates a new folder.

7. rmdir ‚Äì Remove empty directory

Command: rmdir logs

Explanation: Deletes an empty directory.

8. touch ‚Äì Create file

Command: touch test.txt

Explanation: Creates an empty file.

9. nano ‚Äì Edit file

Command: nano test.txt

Explanation: Opens a terminal-based text editor.

10. cat ‚Äì View file content

Command: cat test.txt

Output: hello linux

Explanation: Displays file content.

11. cp ‚Äì Copy file

Command: cp test.txt copy.txt

Explanation: Copies one file to another.

12. mv ‚Äì Move / Rename file

Command: mv copy.txt final.txt

Explanation: Renames or moves files.

13. rm ‚Äì Delete file

Command: rm final.txt

Explanation: Removes a file permanently.

14. rm -r ‚Äì Delete directory recursively

Command: rm -r data

Explanation: Deletes a directory and its contents.

15. chmod ‚Äì Change permissions

Command: chmod 755 test.txt

Explanation: Changes file access permissions.

16. chown ‚Äì Change ownership

Command: sudo chown ubuntu:ubuntu test.txt

Explanation: Changes file owner.

17. whoami ‚Äì Current user

Command: whoami

Output: ubuntu

Explanation: Displays logged-in user.

18. ps -ef ‚Äì Running processes

Command: ps -ef

Explanation: Shows all running processes.

19. top ‚Äì Live system monitor

Command: top

Explanation: Displays CPU and memory usage live.

20. kill ‚Äì Stop process

Command: kill 1234

Explanation: Terminates a running process.

21. df -h ‚Äì Disk usage

Command: df -h

Explanation: Shows disk usage in readable format.

22. free -h ‚Äì Memory usage

Command: free -h

Explanation: Displays RAM usage.

23. grep ‚Äì Search inside files

Command: grep JAVA_HOME ~/.bashrc

Explanation: Searches for text inside files.

24. history ‚Äì Command history

Command: history

Explanation: Shows previously executed commands.

25. ssh ‚Äì Connect to machine

Command: ssh localhost

Explanation: Connects to local or remote Linux machine.
