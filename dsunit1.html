‚òÅÔ∏è üå•Ô∏è Introduction to Cloud Computing

Tip: Remember ‚Äúon-demand + pay-as-you-go + global access‚Äù.

Cloud computing means accessing computing resources (servers, storage, databases, networking, analytics, etc.) over the internet. Organizations use cloud to avoid buying and maintaining physical hardware and to scale systems quickly as demand changes.

Key Concepts

  • On-demand access: resources provisioned instantly.
  • Pay-as-you-go: only pay for what you use.
  • Global accessibility: reachable from anywhere.
  • Shared infrastructure: multi-tenant resource pooling.

‚òÅÔ∏è ‚öôÔ∏è Characteristics of Cloud Computing

Tip: Elasticity and scalability are not the same ‚Äî elasticity is automatic scaling, scalability is capacity to grow.
  • Elasticity: auto-scale resources up/down.
  • Scalability: design to support growth.
  • Resource Pooling: providers share infrastructure.
  • Measured Service: monitoring & billing.
  • Broad Network Access: devices & locations.

‚òÅÔ∏è üèóÔ∏è Cloud Deployment Models

Tip: Use Private for sensitive data, Public for scale & cost savings, Hybrid for best of both.

Deployment models describe how cloud services are delivered and controlled.

  • Public Cloud: services offered by providers (AWS/Azure/GCP).
  • Private Cloud: dedicated infrastructure for one org.
  • Hybrid Cloud: mix of both; enables workload portability.
  • Community Cloud: shared by similar organizations.

‚òÅÔ∏è üß© Cloud Service Models

Tip: IaaS = raw infra, PaaS = platform for devs, SaaS = finished apps.

Service models define the level of abstraction:

  • IaaS: virtual machines, networking, storage.
  • PaaS: managed runtimes, databases, dev platforms.
  • SaaS: delivered software like email, CRM, office suites.

  • Software-defined networking (SDN), NFV

‚òÅÔ∏è üóÑÔ∏è Cloud Services & Platforms

Cloud providers supply many managed services that developers use:

  • CDN: content distribution
  • Analytics: data lakes, streaming, ML platforms

  • Open-source private cloud options: OpenStack, CloudStack

1. What is cloud computing in simple words?

Cloud computing is accessing computing resources (servers, storage, applications) over the internet without owning the physical hardware.

2. What is the difference between scalability and elasticity?

Scalability is the ability to increase capacity to handle growth. Elasticity means automatically adding/removing resources based on demand.

3. What are the three main cloud service models?

IaaS (Infrastructure as a Service), PaaS (Platform as a Service), SaaS (Software as a Service).

4. What role does virtualization play in cloud computing?

Virtualization allows multiple virtual machines to run on a single physical machine, improving utilization and enabling isolation.

5. Why is cloud important for AI/ML?

AI/ML needs elastic compute (GPUs/TPUs), distributed data stores, and scalable inference ‚Äî all of which cloud platforms provide on demand.
