How Big Data is Processed at Scale

When data volume grows beyond the capacity of a single machine, computation must also be distributed. Big Data computation models define how tasks are broken down, executed in parallel, and combined to produce results efficiently.

This phase introduces parallel processing concepts and explains why MapReduce became the foundation of Hadoop, and why newer engines like Spark later evolved.

üìò Exam Insight: Big Data computation models enable parallel processing of massive datasets.

  • Data Parallelism: Same operation applied to different data partitions

AI/ML Context: Model training uses data parallelism to process large datasets faster.
üìò Exam Line: Parallel processing improves performance by dividing work across multiple nodes.

1Ô∏è‚É£3Ô∏è‚É£ MapReduce Model

MapReduce is a programming model that processes large datasets by dividing computation into two main phases: Map and Reduce.

  • Mapper: Processes input data and generates key-value pairs
  • Reducer: Aggregates values for each key
  • Shuffle & Sort: Groups mapper outputs by key
  • Combiner: Performs local aggregation
  • Job Execution Flow: Map ‚Üí Shuffle ‚Üí Reduce
AI/ML Context: Feature aggregation and preprocessing can be expressed using MapReduce logic.
üìò Exam Line: MapReduce is the core processing model of Hadoop.

1Ô∏è‚É£4Ô∏è‚É£ Limitations of MapReduce

  • Disk I/O Overhead: Intermediate results written to disk
  • Not Iterative-Friendly: Inefficient for ML algorithms
  • High Latency: Not suitable for real-time processing
Why Spark? Spark was created to overcome these limitations using in-memory processing.
üìò Exam Line: MapReduce is not suitable for iterative and low-latency workloads.

1Ô∏è‚É£5Ô∏è‚É£ DAG-Based Processing

DAG-based processing represents computation as a Directed Acyclic Graph where each node is an operation and edges represent data flow.

  • Directed Acyclic Graph: Defines execution order
  • Lazy Evaluation: Execution delayed until action is triggered
  • Pipeline Execution: Multiple steps executed together
AI/ML Context: DAGs optimize complex ML pipelines by reducing unnecessary computations.
üìò Exam Line: DAG-based processing is the backbone of Apache Spark.
