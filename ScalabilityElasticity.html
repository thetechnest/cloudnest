Scalability and Elasticity are fundamental cloud characteristics. Every AI/ML pipeline, data platform, and modern web application depends on these two concepts to handle growth, peak loads, and cost optimization.

Scalability is the ability of a system to handle increased workload by adding more resources without affecting performance.

Simple Analogy: Adding more classrooms when the number of students increases.
  • Plan for long-term growth
  • Handles increasing users or data
  • Resources are added deliberately
üìò Exam Line: Scalability is the ability to handle growing workloads by adding resources.

Types of Scalability

üîπ Vertical Scalability (Scale Up)

Increasing the capacity of an existing resource (CPU, RAM).

Example: Upgrading a VM from 4 GB RAM to 16 GB RAM

üîπ Horizontal Scalability (Scale Out)

Adding more instances or servers to distribute workload.

Example: Adding more EC2 instances behind a load balancer

2Ô∏è‚É£ What is Elasticity?

Elasticity is the ability of a system to automatically increase or decrease resources based on real-time demand.

Simple Analogy: Stretching and shrinking a rubber band as needed.
  • Automatic and dynamic
  • Handles sudden traffic spikes
  • Optimizes cost
üìò Exam Line: Elasticity automatically adjusts resources based on demand.

Scalability vs Elasticity
