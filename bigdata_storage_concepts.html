Why Storage Design Changes in Big Data

After moving from traditional databases to Big Data systems, the first major shift I experienced was in storage design. In Big Data, storage is no longer centralized, expensive, or tightly coupled with computation. Instead, it is distributed, fault-tolerant, and built on commodity hardware.

These storage concepts directly lead to Hadoop HDFS and form the physical foundation of modern data lakes used in Data Science and AI systems.

üìò Exam Insight: Big Data storage systems are distributed, scalable, and fault tolerant.

  • Scale-out Storage: Data is spread across many machines instead of one large server

  • Fault Tolerance: Data is replicated to handle failures
AI/ML Context: Large training datasets are stored across nodes, allowing parallel access during model training.
üìò Exam Line: Distributed storage systems provide scalability and fault tolerance using commodity hardware.

9Ô∏è‚É£ Data Lake Concept

A data lake is a centralized repository that stores data in its raw form until it is needed. Unlike traditional data warehouses, data lakes do not enforce a fixed schema at ingestion time.

  • Raw vs Processed Data: Raw data is preserved for future analysis
  • Schema-on-Read: Structure is applied when data is read
  • Schema-on-Write: Structure is applied before storage (traditional DB)
AI/ML Context: Data lakes enable experimentation, feature engineering, and model re-training.
üìò Exam Line: HDFS acts as the physical storage layer of a data lake.
