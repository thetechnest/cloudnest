Data Storage Approaches

Expert View ‚Äì Why Storage Design Matters

While designing cloud applications, choosing the right data storage approach is critical. Storage decisions directly affect performance, scalability, reliability, and cost.

üìò Exam Insight: Storage selection is a key architectural decision in cloud systems.

Classification of Data Storage Approaches

  • Block Storage
  • File Storage
  • Object Storage
  • Relational Databases
  • NoSQL Databases

Block Storage

Block storage stores data in fixed-size blocks and is commonly used for high-performance applications such as databases and virtual machines.

Examples: AWS EBS, Azure Managed Disks, GCP Persistent Disks

File Storage

File storage organizes data in a hierarchical structure using files and folders. It is suitable for shared access scenarios.

Examples: AWS EFS, Azure Files, Google Filestore

Object Storage

Object storage stores data as objects with metadata and unique identifiers. It is highly scalable and cost-effective for unstructured data.

Examples: Amazon S3, Azure Blob Storage, Google Cloud Storage
üìò Exam Line: Object storage is ideal for unstructured data and data lakes.

Relational and NoSQL Databases

Databases store structured and semi-structured data with indexing and query capabilities.

  • Relational: RDS, Azure SQL, Cloud SQL
  • NoSQL: DynamoDB, Cosmos DB, Firestore

Storage Approaches in AWS, Azure, and GCP

  • AWS: S3, EBS, EFS, RDS, DynamoDB
  • Azure: Blob Storage, Disks, Files, Azure SQL, Cosmos DB
  • GCP: Cloud Storage, Persistent Disk, Filestore, Cloud SQL, Firestore

Points to Ponder

  • Is low latency or high throughput more important?
  • Is the data structured or unstructured?
  • How often is data accessed?
  • What are the durability and availability requirements?
  • Is multi-region replication required?
